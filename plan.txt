================================================================================
          PROJECT PLAN: ELEGANT CYBER-TYPING MULTIPLAYER GAME (ZTYPE RECREATION)
================================================================================

1. OVERVIEW
   Recreate ZType with a premium, elegant, and minimal cyber-aesthetic.
   Features include:
   - Solo and 3-Player Team modes.
   - Profile customizer (Username & ship color selection).
   - Real-time global online leaderboard (active players only, resets on tab close).
   - Parallax scrolling background with drifting space debris/towers.
   - Level/wave progression (+25% harder in team mode).
   - Dynamic boss fight with color-sequence typing and letter-bullet defense.
   - Modern canvas-based graphics (glow effects, vector lines, smooth animations).
   - Audio effects downloaded from original resources.

2. ARCHITECTURE
   - Frontend: Vite + React + Vanilla CSS (UI and state management).
   - Game Loop: HTML5 Canvas drawing procedural, glowing ships and objects.
   - Backend WebSocket Server: Node.js (Express + ws) for rooms and leaderboard.
   - Deployment: Configured for Render/Vercel (Client) and Render (Server).

3. DETAILED IMPLEMENTATION PHASES

   PHASE 1: ENVIRONMENT & ASSETS SETUP
   - Initialize Vite + React project in the current directory.
   - Write a Node script to download the original ZType sound assets (.ogg files)
     and place them in the public folder.
   - Create a clean directory structure (src/components, src/game, server/).
   - Delete/cleanup the temporary original_game and zip files.

   PHASE 2: GLOBAL WEBSOCKET SERVER (server.js)
   - Setup an Express + ws server.
   - Implement room creation (4-character code) and lock lobby at 3 players.
   - Maintain a global map of all connected sockets with usernames, current scores,
     and game levels.
   - Broadcast global leaderboard updates to all online users whenever a score
     changes or a player disconnects.
   - Manage real-time room gameplay messaging (bullet cancellation, laser fires,
     enemy health synching, game over).

   PHASE 3: PROFILE CUSTOMIZATION & REAL-TIME LEADERBOARD UI
   - Implement the Main Menu, Profile Customizer (color selector for ship), and Lobby.
   - Create the Leaderboard modal/overlay fetching live data from the socket.
   - Add responsive CSS with HSL variables, dark mode styling, and glassmorphism.

   PHASE 4: PROCEDURAL GAME GRAPHICS & ENGINE
   - Develop HTML5 Canvas game wrapper in React.
   - Build a custom procedural renderer for ships, lasers, and explosions using
     glowing paths and shapes.
   - Create a parallax scenery generator for stars, moons, and side-drifting space
     debris/towers to give a deep spatial perspective.
   - Implement high-fidelity VFX: particle emitters, impact flash-frames,
     and wave transition banners.

   PHASE 5: GAMEPLAY ENGINE & DICTIONARY
   - Implement a text-matching typing queue.
   - Spawning logic for common drones, elite interceptors, and bullet-shooting cruisers.
   - Spawning logic for the boss with dynamic multi-colored phrase cycling.
   - Add single-letter bullet objects that players can destroy.

   PHASE 6: MULTIPLAYER SYNCHRONIZATION
   - Bind keyboard typing to socket messages.
   - Sync targeted enemies and broadcast laser firing coordinates.
   - Synchronize level/wave transitions and coordinates of color-coded words.

   PHASE 7: TESTING & POLISH
   - Verify room locking and color assignments.
   - Check performance on high-refresh screens.
   - Build for production (Client + Server).
================================================================================
